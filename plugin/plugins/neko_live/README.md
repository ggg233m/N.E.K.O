# NEKO Live（neko_live）

> 维护提醒：插件启动会自动注入直播锐评语境；开发者模式开启时再叠加调试语境；插件关闭会发送恢复语境，让猫猫停止把后续普通聊天当成直播弹幕或头像锐评。所有 NEKO 输出链路都必须走 `NekoDispatcher`。

`neko_live` 的产品名是 **NEKO Live**。它已经从“新观众头像锐评”扩展为独立直播互动插件：支持 B 站与实验性抖音只读接入、普通弹幕接话、礼物 / SC / 舰长短句致谢、开场暖场、冷场陪播、主动营业、本场数据和安全画像。首评锐评仍是一个功能模块，不再代表整个产品。

## 当前重点

- 新版面板采用四个普通子界面：**控制台 / 直播间互动 / 观众 / 设置**；开发者模式开启后额外显示独立的“开发者工具”子界面。正常开播只需在控制台完成账号、直播间确认和开始 / 结束直播。
- 直播间配置支持房号或直播链接，并要求“查询直播间 → 核对结果 → 确认直播间”两步完成；底部主按钮只有在账号与直播间准备完成后才可使用。
- B 站扫码登录凭据加密存本机，并用于头像抓取、弹幕连接和直播间查询。B 站无账号连接仅是当前会话显式启用的受限兜底；抖音仍要求导入并校验 cookie。注销只删除本地凭据，暂不做发弹幕 / 私信等写能力。
- 开始直播后，真实观众事件统一经过 EventBus、选择窗口、pipeline、安全门和 `NekoDispatcher`；爆量房间在冷却窗口内按价值（舰长 / SC / 粉丝牌 / 等级 / 长文本）择优。
- 观众档案本地 JSON 持久化：落本机插件默认目录下的 `viewer_profiles.json`。个性化记忆默认开启，只学习安全派生偏好、不保存原始弹幕，90 天未互动会惰性清理；关闭个性化不会破坏基础档案、首次出场判断或本场防复读。普通用户可清空全部档案，或对单 UID 重置印象 / 删除档案。自定义存储位置当前仍不向主播开放。
- B 站 UID / 昵称 / 头像解析，含头像形态 META（默认头像 / 动图 / 挂件装扮）。
- 弹幕锐评请求构造：自适应焦点（昵称与头像哪个更有料就主打哪个，看不到的头像不脑补），并通过 NEKO 当前人设输出。
- 安全测试态（`dry_run`）：产品默认关闭，避免普通用户在不了解该概念时误以为插件已经真实输出。开发者或试播人员需要无声验链时应主动开启；开启后整条 pipeline 照常跑（身份解析、头像抓取、锐评 prompt 构造），但不真的让猫猫开口。
- 限流（`rate_limit_seconds` 即冷却秒数）：直播态下控制最小锐评间隔，避免爆量房间猫猫连珠炮；`0` 关闭限流；开发者沙盒不受限流。
- 启动时注入轻量锐评语境，让猫猫知道这是直播间弹幕/头像锐评场景；实际锐评仍由事件 `respond` 触发，输出要自然、短句、适合 TTS 播放。
- 开发者沙盒调试，沙盒结果独立于直播总结和观众档案。
- 直播安全门、队列保护和降级记录。

账号兜底已在 runtime 边界强制校验：B 站未登录时，只有本次 `connect_live_room` 明确携带 `allow_accountless=true` 才能进入受限连接；该状态不会写入配置，抖音也不接受此兜底。当前离线收尾已经完成文档基线、明确架构债以及礼物调度器和面板弹窗状态修复。真实 30 分钟独播与打包版 UI 验收仍是发布门槛，但维护者已于 2026-07-15 暂缓真机测试；旧 `bilibili_danmaku` 的 47 个公开入口与内部能力迁移矩阵已完成，当前进入获批迁移与退役前置项处理阶段。

## 沙盒数据规则

- 开发者模式是总控开关；关闭时沙盒查询、模拟弹幕、内置案例和聊天开发者工具不可用，但清空沙盒记录仍可用。
- 开发者沙盒只保留运行时临时记录，插件重启或点击“清空沙盒记录”后即遗忘。
- 沙盒查询和模拟弹幕不写观众档案、不进入直播总结、不保存头像 bytes 或 base64 data URL。
- 沙盒模拟弹幕统一走 `DeveloperSandbox -> pipeline -> NekoDispatcher -> push_message -> proactive_message`，并默认投递给 Hosted UI 当前猫猫。

## 当前不做

- 不发送 B 站弹幕、私信或动态。
- 不抓主页资料、贡献值、进房累计。
- 不执行浏览器或键鼠自动化。
- 不整体复制 `bilibili_danmaku` / `bilibili_dm` 的大文件实现；复用旧能力时只拆成小模块软适配（如已吞并的 `DanmakuListener`）并补测试。

## 输出链路提醒

头像只作为本次 `push_message` 的临时视觉输入。发送前必须经过 `NekoDispatcher` 压缩到 message plane 可接受的内联预算内；如果仍然过大，应降级为纯文字锐评，避免 UI 显示 queued 但主程序没有收到 `proactive_message`。
